# 물류관리 시스템 메인 화면 PRD

## 1. 개요

### 1.1 문서 정보
- **문서명**: 물류관리 시스템 메인 화면 PRD
- **버전**: 1.0
- **작성일**: 2025년 7월 18일
- **범위**: 메인 화면 레이아웃 및 네비게이션 기능

### 1.2 목표
사용자가 직관적으로 물류 업무를 수행할 수 있는 효율적인 메인 화면 구조 설계

## 3. 탭 기능 요구사항

### 3.1 탭 생성 규칙
- **트리거**: 좌측 사이드바 메뉴 클릭
- **최대 개수**: 10개
- **중복 방지**: 동일 메뉴 재클릭 시 해당 탭으로 이동
- **제한 초과**: 10개 초과 시 "최대 10개의 탭까지 열 수 있습니다" 경고

### 3.2 탭 표시 방식
- **기본 탭**: 대시보드 (닫기 불가)
- **탭 순서**: 클릭 순서대로 우측에 추가
- **탭 너비**: 최소 120px, 최대 200px
- **오버플로우**: 스크롤 버튼으로 탭 이동

### 3.3 탭 상호작용
- **활성화**: 탭 클릭 시 해당 콘텐츠 표시
- **닫기**: 각 탭의 × 버튼 클릭
- **상태 유지**: 탭 전환 시 이전 입력/스크롤 상태 보존

### 3.4 탭 헤더 영역
- **높이**: 52px (고정)
- **배경색**: #ffffff
- **구성**: 탭 목록 + 스크롤 버튼 (필요시)

```
┌─────────────────────────────────────────────────────────────┐
│ [📊 대시보드] [📦 재고관리 ×] [🚚 배송관리 ×] ... [◀] [▶] │
└─────────────────────────────────────────────────────────────┘
```

### 3.5 탭 디자인 스펙
- **활성 탭**: 하단 테두리 2px (#3b82f6), 배경색 #ffffff
- **비활성 탭**: 배경색 #f8fafc, 호버 시 #e5e7eb
- **탭 간격**: 2px
- **아이콘 크기**: 16px × 16px
- **닫기 버튼**: 14px × 14px, 호버 시 배경색 #fef2f2

## 4. 반응형 동작

### 4.1 화면 크기별 동작
- **1200px 이상**: 전체 레이아웃 표시
- **768px - 1199px**: 사이드바 축소 (아이콘만 표시)
- **767px 이하**: 사이드바 오버레이 모드

### 4.2 모바일 최적화
- **햄버거 메뉴**: 사이드바 토글 버튼
- **탭 스크롤**: 터치 스크롤 지원
- **메뉴 터치**: 터치 영역 최소 44px

## 5. 상태 관리

### 5.1 필수 상태 (탭 시스템)
```javascript
{
  // 현재 활성화된 탭
  activeTab: 'dashboard',
  
  // 열린 탭 목록
  openTabs: [
    { id: 'dashboard', title: '대시보드', icon: 'Home' },
    { id: 'inventory', title: '재고관리', icon: 'Package' },
    { id: 'shipping', title: '배송관리', icon: 'Truck' }
  ],
  
  // 사이드바 상태
  sidebarOpen: true,
  
  // 각 탭별 상태 보존
  tabStates: {
    dashboard: { scrollPosition: 0, formData: {}, activeSubTab: 'overview' },
    inventory: { scrollPosition: 100, formData: {}, selectedItems: [] },
    shipping: { scrollPosition: 50, formData: {}, selectedRoute: null }
  },
  
  // 탭 헤더 상태
  tabHeader: {
    scrollPosition: 0,  // 탭 목록 스크롤 위치
    showScrollButtons: false  // 스크롤 버튼 표시 여부
  }
}
```

### 5.2 상태 변경 액션 (탭 시스템)
- `openTab(menuItem)`: 새 탭 열기
- `closeTab(tabId)`: 탭 닫기
- `setActiveTab(tabId)`: 활성 탭 변경
- `toggleSidebar()`: 사이드바 토글
- `scrollTabs(direction)`: 탭 목록 스크롤
- `updateTabState(tabId, state)`: 탭 상태 업데이트

## 6. 사용자 경험 (UX) 요구사항

### 6.1 네비게이션 흐름 (탭 시스템)
1. 사용자가 좌측 사이드바 메뉴 클릭
2. 시스템이 해당 탭의 존재 여부 확인
3. 기존 탭이 있으면 해당 탭으로 이동
4. 새 탭이면 탭 개수 확인 후 생성 (최대 10개)
5. 탭 헤더에 새 탭 추가
6. 메인 콘텐츠 영역에 해당 화면 표시

### 6.2 탭 관리 UX
- **탭 생성**: 새 탭은 기존 탭 우측에 추가
- **탭 활성화**: 클릭된 탭의 하단 테두리 강조 표시
- **탭 닫기**: × 버튼 클릭 시 해당 탭 제거
- **탭 스크롤**: 탭이 10개 가까이 되면 좌우 스크롤 버튼 표시
- **상태 보존**: 탭 전환 시 이전 스크롤 위치 및 입력 데이터 유지

### 6.3 시각적 피드백 (탭 시스템)
- **메뉴 호버**: 사이드바 메뉴 호버 시 배경색 변경 (#e5e7eb)
- **활성 탭**: 하단 테두리 강조 (#3b82f6), 배경색 #ffffff
- **비활성 탭**: 배경색 #f8fafc, 호버 시 #e5e7eb
- **탭 닫기 버튼**: 호버 시 배경색 #fef2f2
- **로딩 상태**: 탭 전환 시 로딩 스피너 표시
- **오류 상태**: 탭 로딩 실패 시 에러 메시지 표시

## 7. 기술적 구현 요구사항
